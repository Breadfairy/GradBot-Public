# Scripts

Repo shell entrypoints live in `scripts/`.

Typical shell:

```bash
cd scripts
```

## Primary Entry Points

### `./run_tune.sh PROFILE LABEL [KEY=VALUE ...]`

Purpose:

- main tune entrypoint
- validates the profile
- ensures local klines exist
- writes `causality-audit.json` and fails before tuning if an active
  tune/holdout overlap or active future alignment is detected
- builds `host-spec/`
- runs `build/native/gradbot_tune`
- runs Python post handling
- runs selected-config holdout trace through Python orchestration

Anchor behavior:

- if no anchor env var is set, the run uses current data and stamps the
  runtime anchor into the run metadata
- pass `TUNE_ANCHOR_DATE=YYYY-MM-DD` to run a historical tune/holdout
  window using a UTC date anchor
- `TUNE_ANCHOR_MS=<utc_ms>` still works, but the date form is the
  preferred user-facing input
- `src/data/prepare_klines.py` expands local kline fetch far enough back to
  cover the anchored window

Writes:

- `outputs/tuning/<LABEL>/`

Delegates to:

- `src/tune/run.py`
- `src/data/prepare_klines.py`
- `src/data/causality_audit.py`
- `src/tune/host.py`
- `build/native/gradbot_tune`
- `src/tune/post.py`
- `src/tune/trace.py`

Flash sweeps:

- pass `TUNE_FLASH=1` to disable tune charts and set generated configs'
  `CHARTS_TIMEVAL` and `CHARTS_TRADES` false for fast no-PNG smoke runs

Standalone backtest scripting is retired. Use `run_tune.sh` with a scalar
one-combination profile for new single-config work.

## Kline Maintenance

### `./get_klines.sh TICKER INTERVAL [DAYS]`

Purpose:

- create or extend a local kline file in `inputs/klines/`

Delegates to:

- `src/data/prepare_klines.py get`

### `./update_klines.sh`

Purpose:

- forward-fill all local kline files to now

Delegates to:

- `src/data/prepare_klines.py update`

## Posture Modeling

Posture clustering is model/artifact generation, not a normal tune or live
runtime. It has one script entrypoint because generated posture artifacts feed
both tuning and live.

### `scripts/run_clustering.sh [CONFIG]`

Purpose:

- builds causal rolling-window clustering features
- writes posture label CSVs for tuning `DAILY_CLUSTER_PATH`
- writes `model/cluster_model.json` artifacts that can be promoted into
  `inputs/live/model/` for live posture inference and C-host tune inference

Default:

- `inputs/clustering/current/linkusdt-6h-ema-fast-posture.json`

Delegates to:

- `src/clustering/run_cluster.py`

Direct clustering experiment helpers remain under `src/clustering/`. They
are not part of tune/live runtime unless a generated artifact is deliberately
selected in a profile or promoted into `inputs/live/model/`.

## Live Deployment Entry Points

Run from the repo root:

```bash
scripts/run_install.sh
scripts/run_live.sh
```

### `scripts/run_install.sh`

Purpose:

- creates `.venv/live`
- installs the pinned `requirements/live.txt`
- checks that `inputs/profiles/user/live-config.json` and
  `inputs/live/model/cluster_model.json` are present
- installs or checks for `tmux` when Homebrew or apt is available

### `scripts/run_live.sh`

Purpose:

- starts the live runtime in a `tmux` session named `gradbot-live`
- calls `scripts/run_install.sh` automatically when the live environment or
  required packages are missing
- uses `caffeinate` on macOS when available
- uses `systemd-inhibit` on Linux when available
- runs `python3 -m live.live` from the live virtual environment

Live runtime behavior starts in `src/live/live.py`. On launch it fetches
fresh candles, validates the warmup context, restores `*_state.csv` when the
symbol/interval/mode match, and then waits for the next closed 1h candle.
For each new closed 1h candle, it appends one row to `*_decisions.csv`.
Trade rows in `*_trades.csv` include the same core decision context.

## Analysis Tools

These are direct Python commands from the repo root:

```bash
python3 src/tools/marker_audit.py --profile PROFILE --anchor-date YYYY-MM-DD
python3 src/tools/lock_sweep.py --profile PROFILE --out OUT_DIR
PYTHONPATH=src python3 src/tools/control_error_audit.py \
  --profile PROFILE --out OUT_DIR --window holdout
```

Most tools support `--window holdout` or `--window tune`; holdout is the
default because it matches tuned-profile validation.
`control_error_audit.py` writes macro required-move step-error candidates and
compares current flags with a positive-gradient step-error replay.
The old standalone PID research overlays were retired after peak-lock/PID
logic moved into `src/strategy/supervisor.py` and `src/tune/peak_lock.py`.

## Agent Rule

If you are tracing runtime behavior, start from the shell script that launched
the run, then follow the Python entrypoint it calls. Do not jump straight into
random modules in `src/`.
